QUICK START:
================================================================================

1. Navigate to the project directory:
   $ cd /Users/garfenter/development/products/erp-crm/twenty

2. Run the one-click deployment script:
   $ ./garfenter-start.sh

3. Wait for services to start (1-2 minutes on first run)

4. Access the CRM in your browser:
   http://localhost:3000

5. Create your first admin account and start using Garfenter Clientes!

SYSTEM REQUIREMENTS:
================================================================================

- Docker Desktop 20.10+ (with Docker Compose V2)
- 4 GB RAM minimum (8 GB recommended)
- 10 GB free disk space
- Available ports: 3000, 5432, 6379

IMPORTANT SECURITY NOTES:
================================================================================

FOR PRODUCTION DEPLOYMENT:

1. Change the database password in .env.garfenter:
   PG_DATABASE_PASSWORD=your_secure_password

2. Generate and set a secure APP_SECRET:
   $ openssl rand -base64 32
   APP_SECRET=generated_secret_here

3. Update SERVER_URL to your production domain:
   SERVER_URL=https://crm.garfenter.com

4. Configure SSL/TLS using a reverse proxy (nginx, traefik)

5. Set up automated backups

COMMON COMMANDS:
================================================================================

# View logs
docker compose -f docker-compose.garfenter.yml logs -f

# Stop services (keep data)
docker compose -f docker-compose.garfenter.yml stop

# Start services
docker compose -f docker-compose.garfenter.yml start

# Restart services
docker compose -f docker-compose.garfenter.yml restart

# Stop and remove (keep data)
docker compose -f docker-compose.garfenter.yml down

# Backup database
docker exec garfenter-postgres pg_dump -U postgres garfenter_clientes > backup.sql

# Check status
docker compose -f docker-compose.garfenter.yml ps

DATA PERSISTENCE:
================================================================================

All data is stored in Docker volumes that persist across container restarts:

- garfenter-db-data: PostgreSQL database (all CRM data)
- garfenter-redis-data: Redis cache (sessions, queues)
- garfenter-server-storage: File uploads and attachments
